
## Комплексная лабораторная работа

#### Этап 1. Системный анализ предметной области (на основе ЛР 1)

**Задачи:**
- [ ] 1. Выбрать предметную область и обосновать выбор.
- [ ] 2. Провести системный анализ: выделить цели системы, подсистемы, элементы и связи между ними.
- [ ] 3. Построить морфологическое описание системы (состав и структура).
- [ ] 4. Построить функциональное описание системы (основные функции и процессы).
- [ ] 5. Составить дерево целей и провести декомпозицию главной цели до 3‑го уровня.

**Результат:** отчёт с описанием системы, схемы структуры и функций, дерево целей.
#### Этап 2. Функциональное моделирование (на основе ЛР 5)

**Задачи:**

- [ ] 1. Построить контекстную диаграмму IDEF0 для системы.
- [ ] 2. Провести декомпозицию функций до 3‑го уровня детализации.
- [ ] 3. Для каждой функции описать:

	- входы (данные/объекты, поступающие на вход);
    - выходы (результаты выполнения функции);
    - управления (правила, политики, стандарты);
    - механизмы (ресурсы, исполнители).
- [ ] 4. Проанализировать типы связности между функциями.

**Результат:** иерархия IDEF0‑диаграмм, описание функций и их параметров.

#### Этап 3. Моделирование потоков данных (на основе ЛР 2)

**Задачи:**

- [ ] 1. На основе функциональной модели построить контекстную DFD.
- [ ] 2. Детализировать процессы до уровня элементарных функций (уровни 0, 1, 2).
- [ ] 3. Описать потоки данных, хранилища и внешние сущности.
- [ ] 4. Проверить модель на полноту и согласованность с IDEF0.

**Результат:** набор DFD‑диаграмм (уровни 0–2), словарь потоков данных.

#### Этап 4. Моделирование данных (на основе ЛР 3)

**Задачи:**

- [ ] 1. Выделить сущности, атрибуты и связи в предметной области.
- [ ] 2. Построить ER‑диаграмму в нотации Баркера или IDEF1X.
- [ ] 3. Нормализовать модель данных до 3НФ.
- [ ] 4. Описать типы связей (идентифицирующие, неидентифицирующие) и кардинальность.
- [ ] 5. Сопоставить ER‑модель с потоками данных из DFD.

**Результат:** ER‑диаграмма, словарь сущностей и атрибутов, описание связей.

#### Этап 5. Объектно‑ориентированное моделирование (на основе ЛР 4)

**Задачи:**

- [ ] 1. Выделить классы, атрибуты и методы на основе предметной области и ER‑модели.
- [ ] 2. Построить диаграмму классов (Class Diagram).
- [ ] 3. Добавить диаграмму прецедентов (Use Case Diagram) для ключевых пользователей.
- [ ] 4. Для 2-3 ключевых сценариев построить диаграммы последовательностей (Sequence Diagram).
- [ ] 5. Обеспечить согласованность UML‑диаграмм с функциональными и информационными моделями.

**Результат:** набор UML‑диаграмм (классы, прецеденты, последовательности), описание классов и сценариев.

#### Этап 6. Моделирование бизнес‑процессов (на основе ЛР 6)

**Задачи:**

- [ ] 1. Выделить ключевой бизнес‑процесс в предметной области (например, «Выдача книги читателю»).
- [ ] 2. Построить BPMN‑диаграмму процесса с детализацией до подпроцессов.
- [ ] 3. Описать роли, события, шлюзы, потоки данных и артефакты.
- [ ] 4. Проанализировать возможности оптимизации процесса (сокращение времени, устранение дублирования и т. д.).
- [ ] 5. Сравнить BPMN‑модель с DFD и IDEF0, выявить расхождения.

**Результат:** BPMN‑диаграмма, описание процесса, рекомендации по улучшению.

#### Этап 7. Применение нечёткой логики (на основе ЛР 7)

**Задачи:**

- [ ] 1. Выбрать задачу с нечёткими условиями (например, «Оценка приоритета обработки запроса читателя»).
- [ ] 2. Определить лингвистические переменные (например, «срочность запроса») и термы (низкий, средний, высокий).
- [ ] 3. Построить систему нечёткого вывода:
	- сформулировать правила нечёткой импликации;
	- выбрать метод дефаззификации.
- [ ] 4. Протестировать систему на 3–5 примерах запросов.
- [ ] 5. Интегрировать модель нечёткой логики в общую архитектуру системы (например, как модуль принятия решений).

**Результат:** модель нечёткой системы, набор правил, результаты тестирования, схема интеграции.

#### Этап 8. Интеграция и валидация модели (на основе ЛР 8)

**Задачи:**

- [ ] 1. Импортировать все созданные модели в CASE‑средство.    
- [ ] 2. Проверить согласованность между диаграммами:
    - соответствие сущностей ER‑модели и классов UML;
    - связь потоков данных DFD с атрибутами сущностей;
    - соответствие функций IDEF0 и процессов BPMN.
- [ ] 3. Выявить и устранить противоречия (например, дублирование функций, несоответствие типов данных).
- [ ] 4. Сформировать единый отчёт по проекту, включающий:
    - описание предметной области;
    - все диаграммы (IDEF0, DFD, ER, UML, BPMN, нечёткая модель);
    - словарь терминов и глоссарий;
    - рекомендации по реализации системы.
- [ ] 5. Подготовить презентацию проекта (5–7 слайдов) с ключевыми результатами.
